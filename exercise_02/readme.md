# Пошук шляхів у графі міст України за допомогою DFS і BFS

## Мета

Реалізувати алгоритми DFS (Depth-First Search) та BFS (Breadth-First Search) для знаходження шляхів між містами України у графі транспортної мережі. Провести порівняння отриманих шляхів та пояснити різницю у результатах.

---

## Використані інструменти

- Python 3  
- NetworkX — для створення графа міст та доріг  
- Matplotlib — для візуалізації графа  

---

## Опис реалізації

1. **Створення графа**  
   - Вершини графа — міста України  
   - Ребра графа — дороги між містами з вагою, що відповідає відстані у кілометрах  

2. **Реалізація алгоритмів**  
   - DFS (Depth-First Search) — ітеративний пошук у глибину  
   - BFS (Breadth-First Search) — ітеративний пошук у ширину  
   - Функція `path_weight` — обчислення сумарної ваги шляху  

3. **Візуалізація графа**  
   - Вершини відображаються як міста  
   - Ребра відображають дороги з підписами ваг  

---

## Результати

Шлях від Києва до Запоріжжя:

| Алгоритм | Отриманий шлях                                   | Сумарна вага (км) |
|----------|-------------------------------------------------|------------------|
| DFS      | Київ → Вінниця → Одеса → Миколаїв → Херсон → Дніпро → Запоріжжя | 1355             |
| BFS      | Київ → Полтава → Дніпро → Запоріжжя             | 625              |

### Пояснення різниці

- DFS обирає перший доступний шлях у глибину, тому не гарантує мінімальну сумарну вагу.  
- BFS досліджує граф пошарово та знаходить шлях із мінімальною кількістю ребер, що у цьому випадку дозволило знайти коротший шлях за сумарною відстанню.  

---

## Висновки

1. Алгоритми DFS та BFS успішно реалізовані для знаходження шляхів у графі міст України.  
2. Результати демонструють, що DFS може знайти довший шлях, тоді як BFS наближено знаходить коротший маршрут.  
3. Візуалізація графа допомагає наочно оцінити зв’язки між містами та маршрути.
